Phase 4 walkthrough — ScanGroup

pyctrl Python front-end migration · work items W1–W9 (complete)

Goal of Phase 4: port ScanGroup — the N-dimensional parameter-sweep layer that turns one parametric sequence definition into the ordered list of concrete seq structs a run produces. A scan file writes g().Pushout.Blue.Freq.scan(1) = [..]; getseq(n) materializes the n-th point; ExpSeq(getseq(n)) then merges it onto the config and serializes. Phase 4 makes that whole front edge run in Python and produce the same per-point bytes as MATLAB.

This doc walks the nine work items, the locked design decisions, the one byte-critical algorithm (column-major expansion), the two MATLAB oracles that pin it, and the test-only surface (usevar, concat, load, the access-tracker lint).

TL;DR

  • THE ONE RULE still governs. ScanGroup itself emits no bytes, but it decides which values get baked into each sequence — so the headline check is that ExpSeq(g.getseq(n)) serializes byte-identical to MATLAB's per-point capture over real YbScans (W6).
  • Faithful 1:1 transliteration of ScanGroup.m + ScanParam/ScanInfo/ScanAccessTracker/EnableScan/MutableRef. The brassboard "visited-tree" framing was declined: a parameter is swept vs fixed structurally (it lives under scan.vars[dim].params vs scan.params), decided at .scan()-call time — no listeners, no visited tree.
  • ScanGroup is a handle (reference) class — no copy-on-assignment. Only the struct param data deep-copies (MATLAB value structs; Python dicts alias), at four boundaries: getfullscan, whole-scan assign, cat_scans, dump/load.
  • The column-major mixed-radix expansion in getseq_in_scan is the #1 byte risk; it is pinned field-for-field, value-for-value, order-for-order against real MATLAB by the W4 oracle and byte-for-byte by the W6 per-point oracle.
  • Full default suite: 391 passed / 13 deselected, no_hardware (engine never loaded; ScanGroup touches no SeqConfig/SeqManager/engine at all).
  • Status: W1–W9 all done. Phase 4 complete. Next is Phase 5 (run loop / abort-pause).

The mental model — where ScanGroup sits

A scan file builds a ScanGroup declaratively, then hands it plus a sequence function to the runner. The runner's inner loop is, in essence:

for n = 1:nseq(g)
    params = g.getseq(n)        # the n-th point: concrete scalar values
    s = ExpSeq(params)          # merge params onto the config consts
    SequenceFn(s)               # build the step tree
    serialize(s)  -> bytes      # hand to libnacs

So ScanGroup's job is purely data-model arithmetic: store the swept/fixed parameter structure, and enumerate it deterministically. Two structural facts drive everything:

The Python DSL — where it diverges from MATLAB

MATLAB overloads subsref/subsasgn for syntax Python can't reproduce (.scan(dim) = vals, g(i) = rhs). The port keeps everything else 1:1 and diverges only where forced:

MATLABpyctrlNote
g(i).a.b = xg(i).a.b = xsame — fixed param

Because scan/assign/size/usevar/toscan are real methods, a parameter literally named one of those is shadowed — MATLAB likewise special-cases scan/usevar. The proxy classes (ScanParam, ScanInfo) fold in the SubProps role: rather than a second generic proxy, each carries the accumulated dotted path itself (g(i) has path=(); g(i).a.b is the same class with path=('a','b')) and forwards (idx, path, …) to the group exactly as MATLAB forwards (idx, S).

The byte-critical core — column-major expansion

This is the one place a subtle bug would silently change which value lands in the bytes. getseq_in_scan(scanidx, seqidx) is a deterministic mixed-radix enumeration: dimension 1 varies fastest, dummy (size-0) dimensions are skipped, indices are 0-based internally (v[subidx]) but 1-based at the API:

scan = getfullscan(scanidx)        # base-merged, independent deepcopy
seq  = scan.params                 # the fixed values
seqidx -= 1                        # 0-based
for var in scan.vars:
    if var.size == 0: continue     # dummy dim
    subidx  = seqidx % var.size
    seqidx  = seqidx // var.size
    for (v, path) in leaves(var.params):
        seq[path] = v[subidx]      # pick this point's value off the axis
return seq

Leaf-walk order matters (it determines field insertion order, which the W4 oracle compares order-sensitively): leaves() is depth-first in dict insertion order, empty sub-structs skipped, a list/array (an axis) is a leaf. Python dicts preserve insertion order, so this matches MATLAB fieldnames order.

Base merge — the other half

getfullscan(idx) merges a scan with its base chain (every scan's baseidx defaults to 0 = the "default" scan, so the default is always merged), with a dirty-flag cache. The child always wins (a base value never lands on a child-owned path, tracked by find_scan_dim), base axes keep their original dimension index, and sizes are recounted with a strict consistency check. It returns an independent deepcopy — the value-copy boundary that lets every caller mutate freely without corrupting the cache.

W1 — EnableScan + MutableRef

The process-local pre-run gate (set at the top of ybStartScan/ybRunScans), not the MemoryMap gentle-abort (that's Phase 5). MATLAB's RAII delete restore is ported as a Python context manager (ctor sets immediately + captures old, __exit__/delete() restore once, disable() cancels, __del__ best-effort backstop); FacyOnCleanup folded in, not ported standalone.

W2 / W3 — the core + materialization

The data model is verbatim: _scans = list of {baseidx, params, vars:[{size, params}]}, plus _base, _runparam (a DynProps), the tri-state _use_var_* trees, and _scanscache dirty flags. Authoring decides swept-vs-fixed at call time; isarray decay (scalar / str / 1-element list → fixed param) and the exact MATLAB error messages ("Cannot scan a fixed parameter.", …) are ported. W3 adds the column-major getseq, the size/dim queries, and ScanInfo — a read-only (value, dim) view where a trailing () resolves the path (info.c() → (val, dim); info.k() → (proxy, -1); info.e(2) → (2, 0) default form).

W7 — concat, load, and the mega dotest

The test-only surface production never uses but TestScanGroup.m does:

  • cat_scans (static) + horzcat + toscan: build a NEW group whose scans are the base-merged scans of every input, baseidx reset to 0, runparam copied from the first input. The Python form of MATLAB's [g1, g2].
  • Multi-index addressing: a full colon g(:) maps to the default (idx 0, the MATLAB magic colon), but a bounded g(slice(2, g.end)) or g([1,3]) yields a ScanParam carrying a list idx — consumed only by cat_scans (no field authoring on it, matching MATLAB's display-only use).
  • load / load_v1 / load_v0 — the inverse of dump, version-dispatched; v1 deep-copies + injects use_var defaults for old payloads, v0 is the legacy p/scan struct-array layout. validate() is a faithful no-op stub (a latent MATLAB bug — load calls it expecting checks that were never written; replicated for behavior parity).
  • The dotest port keeps every semantic assert (build / dump / getseq / get_scan / concat / load / assign battery) and drops every display-only checked_disp line. Key traces it pins: g2 = [g, g] is an independent deepcopy (mutating g via setbase leaves g2.dump() unchanged), and the load→assign battery (g3(3)=g3(1), g3(5)=g3(2), g3(:).name=…, g3(4)=struct).

W8 — usevar (runtime-variable scan axes)

usevar marks a scan axis as a runtime variable (injected as a sequence global per shot) instead of expanding it into distinct compiled sequences — so seqs that differ only in usevar dims share one compiled id. getseq_with_var(n) splits each axis into either a fixed expansion (folded into seq, advancing the mixed-radix id + cross-scan count) or a variable (collected into a vars list of (path, value)).

Which axes are variables is computed_use_var: a dim is a variable iff every leaf under it resolves (via the per-scan merged use_var tree) to exactly +1. The resolution rule (check_use_var) is subtle and ported faithfully: the root node's flag, overridden by the leaf field-node's flag if nonzero — intermediate path nodes are not consulted. (So g().A.C.usevar(true) makes only A.C a variable; A.B stays fixed.) The full TestScanGroup.test_usevar battery (9 states A–I incl. a load round-trip and cache invalidation) is ported.

Latent bug replicated: computed_use_var's cache is keyed only by length(scans), so base-index churn that keeps the length can return stale data. Most mutators clear the cache (masking it); the port keeps the same length-keyed cache for behavior parity rather than "fixing" it.

W9 — ScanAccessTracker (unused-parameter lint)

A post-hoc lint that catches channel-name typos: it consumes the per-sequence accessed tree the Phase-3 DynProps already produces (get_accessed), unions it per scan, and once every sequence of a scan is collected, warns about any fixed/swept parameter no sequence touched. A param counts as used if any sequence in the scan accessed it — so scanning a parameter that disables code in some shots doesn't false-positive. It's pure bool-tree arithmetic:

HelperDoes
to_bool_structparams struct → bool tree (every non-struct leaf → True)
merge_structunion of two trees (a non-struct on either side wins as True)
compute_unusedsubtract accessed from params → only never-accessed leftovers
show_unusedrender the leftover as the dotted, indented warning

Porter trap avoided: MATLAB to_bool_struct "returns" its in-place-mutated input; a naive Python port that builds a fresh out but forgets to return it yields empty trees. The port builds a fresh dict and returns it explicitly.

5 of 6 TestScanAccessTracker methods are ported (test_reset is MATLAB warn-prefix plumbing). MATLAB compares evalc'd warning text with a version-specific prefix; the port instead captures the Python warnings list and compares message bodies directly — prefix-free.

Locked design decisions

  1. No visited tree. Swept-vs-fixed is structural and decided at .scan()-call time; the brassboard (values, visited, fieldname) triple was declined (doubly wrong, and the access tree is zero bytes).
  2. Handle semantics; copy only the data. ScanGroup is a reference class — deepcopy is applied at exactly four boundaries (getfullscan result, whole-scan assign source, cat_scans merged scans, dump/load) because Python dicts alias where MATLAB structs copy.
  3. Scramble is NOT in ScanGroup. Zero RNG here; the point-permutation is a runp value consumed by the runner (Phase 5). Expansion is the deterministic column-major enum, full stop.
  4. Production-first staging. A YbScans survey showed production uses only single-group, fixed params, 1-D/2-D .scan(dim), runp, getseq — never g(n)/setbase/usevar/horzcat/toscan. So W1–W6 delivered the production-sufficient milestone first; W7–W9 are the test-only surface for full parity.
